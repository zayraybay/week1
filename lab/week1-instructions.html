Getting Started with Git and GitHub

We can think of Git like Dropbox, but for code files. Git is the open-source technology that powers it all; GitHub is a commercial hosted Git service – owned by Microsoft. Git repositories are folders on your computer that have magic powers. These powers include:

  • The ability to automatically synchronize files within the folder between your computer and GitHub.com
  • Watching for changes you make to files, then giving you the ability to either move forward with those changes, or revert back
  • Collaborate with others on the same code, without stepping on toes

Creating a Repository

The first step to using Git and GitHub is creating a Git repository. The basic process is as follows:

  1. Go to GitHub.com, make sure you're logged-in.
  2. Click the "New" button next to "Repositories", or from the "plus sign" in the top-right.
  3. Give your repository a name, keep it public, and click "New Repository".
  4. You should see the initial "quick setup" page.
  5. In VSCode, clone the repository the same way we cloned the in-class repository – start with a blank VSCode window, "Clone Repository" -> "Clone from GitHub", and select your main code directory as the location.

Once the repository is created, Git is always watching. If it detects that anything has changed in our repository folder, we'll see those changes in VSCode. We can then repeat the commit-and-push sequence to commit those changes and send them to GitHub.

Your mild-mannered, ordinary folder is now a REPOSITORY and fully synchronized with GitHub! You'll know everything went right if you see your changes at GitHub.com. Remember that your account at GitHub.com is public - if you can see it, I can see it... everyone can see it!

Don't be that guy/gal/non-binary pal

Cloning an existing repository

We did this at the beginning of class, but just to document the process, so we have it for the future:

  1. Open a new, blank VSCode window. File -> New Window is the easiest way. Click the Source Control (SCM) in left-hand sidebar.
  2. "Clone Repository"
  3. "Clone from GitHub" should give you a list of options from your GitHub account. Alternatively, enter GitHub repository URL, e.g. https://github.com/eng/hello-html
  4. Select a folder – find our "code" folder and "Select Repository Location" - this will create a new subfolder with all the code from the repository
  5. Would you like to open the cloned repository? Yes, yes we would. Select "Open"

Committing and Pushing

Once changes are made to your code – either new files are created, or changes are made to existing files, you'll want to commit and push those changes to GitHub. This is a two step process, both done within the "Source Control" tab in VSCode.

  1. Commit – you commit changes by writing a "commit message" – a brief summary of the work that you've done – and hitting the "check mark" icon.
  2. Push – after you've committed your changes, hit the "three dots" icon and select "Push". This will take the changes you've committed and send them up to GitHub.

Building a simple profile page

  1. Create a new repository at GitHub.com. Call it hello-world, clone and open it in VSCode.
  2. Create a new file named index.html.
  3. Build a profile page (i.e. "about me" page or resume) using properly-formed HTML. Make it your own – include images, link to social media accounts, or whatever you like. Use the provided HTML cheatsheet to get an idea of some of the basic HTML elements and how to use them.
  4. Click the "Source Control" icon in the left-hand sidebar and commit and push the changes to GitHub (see "Committing and Pushing" above).
  5. Go to GitHub.com and verify that your code is there and publicly accessible (protip: use "Incognito Mode" to give it a quick test).
  6. Make changes to the page and commit and push again – make sure you're comfortable with this cycle, as we'll be doing this a lot this quarter.
